How to use a free VPN with Clash Verge (Windows / macOS / Linux)

Step-by-step guide to import a free VPN subscription into Clash Verge on Windows, macOS, or Linux. TCP+TLS verified nodes, updated hourly.

🪜 Steps

  1. Install Clash Verge

    Download the latest release from Clash Verge Rev. Pick the installer matching your OS (.msi for Windows, .dmg for macOS, .deb/.rpm/.AppImage for Linux). Run it and launch the app.

  2. Open the Subscriptions panel

    In the left sidebar, click Profiles. You'll see an empty list the first time. Click the + or Import button at the top.

  3. Paste the subscription URL

    Copy the Clash URL from the homepage and paste it into the URL field. Give it any name you like. Click Save. Clash Verge will download the profile in a few seconds.

  4. Enable the profile

    Click the profile card so it shows as active. In the Proxies tab you should now see a selector group with all the free nodes.

  5. Turn on system proxy

    Back on the dashboard, toggle System Proxy or TUN Mode ON. System proxy works for browsers; TUN mode routes all traffic including CLI tools and games — TUN mode requires admin/sudo the first time.

  6. Test it

    Open a browser and go to ipinfo.io. The country should match the node you picked. If it still shows your home country, double-check the proxy toggle and that your browser isn't bypassing system proxy (Firefox has its own proxy settings).

💡 Tips & Troubleshooting

Which node should I pick?

Start with the selector group's auto-select (URL-Test) — it picks the fastest by latency. If a node feels slow, switch to a different country manually.

How do I update the subscription?

Right-click the profile card → Update. Or enable auto-update in the profile settings (we recommend every 1 hour since upstream is refreshed hourly).

Can I use this with browsers only (no TUN)?

Yes. Turn on System Proxy only. Most browsers respect it. Chrome/Edge use system proxy by default; Firefox needs "Use system proxy settings" in its Network settings.

Clash Verge says "profile update failed"

Usually a transient GitHub raw content outage. Retry in a few minutes. If it persists, check that you can open the subscription URL directly in a browser.
