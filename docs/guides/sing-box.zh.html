🪜 操作步骤

  1. 挑一种客户端

    sing-box 有两种形态:命令行二进制 (Windows/macOS/Linux) 和移动端原生 App (iOS/Android)。桌面想要图形界面,装 Hiddify Next。iOS 装免费的 sing-box;安卓去 GitHub Releases 下 APK。

  2. 复制 sing-box 订阅链接

    在首页复制 sing-box 那一行的 URL (结尾是 singbox.json)。

  3. 导入 (图形界面路线)

    打开 Hiddify Next 或 sing-box App:Profiles → Add Profile → 粘 URL。启用 profile,拨 VPN 开关。

  4. 导入 (命令行路线)

    把 JSON 保存到本地:curl -o config.json <订阅 URL>。运行:sing-box run -c config.json。默认在 127.0.0.1:2080/2081 起 SOCKS5 + HTTP 代理。浏览器或系统代理指到这里。

  5. 开启 TUN (可选,CLI 路线)

    想全局接管所有流量,改 config.json 加一个 TUN inbound (参考官方文档)。sing-box 以 root/管理员运行。所有 TCP+UDP 都走所选节点。

  6. 验证

    curl --proxy socks5://127.0.0.1:2080 https://ipinfo.io (CLI) 或直接打开浏览器 (GUI),国家应该匹配所选节点。

💡 常见问题排错

sing-box 和 Clash 选哪个?

sing-box 对新协议 (VLESS+Reality、Hysteria2、TUIC) 原生支持更好,体量更轻。Clash Meta 也支持这些协议,但 sing-box 实现更贴近官方规范。

能自动更新订阅吗?

GUI 里开自动更新,间隔设 1 小时。CLI 加个 cron:0 * * * * curl -o /etc/sing-box/config.json <URL> && systemctl reload sing-box。

macOS 有菜单栏版吗?

用 Hiddify Next 或 ClashX Meta。sing-box 自己暂时没发 macOS 原生 GUI。

安卓没系统代理,sing-box 怎么接管流量?

sing-box 安卓 App 用 VPNService,和 v2rayNG 一样。开关打开就全局生效。
